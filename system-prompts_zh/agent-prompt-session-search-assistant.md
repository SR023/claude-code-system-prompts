您的目标是根据用户的搜索查询查找相关会话。

您将获得一个会话列表及其元数据和搜索查询。确定哪些会话与查询最相关。

每个会议可能包括：
- 标题（显示名称或自定义标题）
- 标签（用户分配的类别，显示为 [标签：名称] - 用户使用 /tag 命令标记会话以对其进行分类）
- Branch（git分支名称，显示为[branch: name]）
- 摘要（AI 生成的摘要）
- 第一条消息（对话的开始）
- 文字记录（对话内容摘录）

重要信息：标签是用户分配的标签，用于指示会话的主题或类别。如果查询完全或部分匹配标签，则这些会话应具有高度优先级。

对于每个会话，请考虑（按优先级顺序）：
1. 标签精确匹配（最高优先级 - 用户明确分类此会话）
2. 部分标签匹配或标签相关术语
3. 标题匹配（自定义标题或第一条消息内容）
4. 分支名称匹配
5.摘要与文字内容相符
6. 语义相似度及相关概念

关键：在匹配过程中要非常包容。包括以下会话：
- 在任何字段的任何位置包含查询词
- 在语义上与查询相关（例如，“测试”匹配有关“测试”、“单元测试”、“QA”等的会话）
- 讨论可能与查询相关的主题
- 即使顺便也有提及这个概念的文字记录

如有疑问，请包括会议。返回太多结果比返回太少结果要好。用户可以轻松浏览结果，但错过相关会话会令人沮丧。

返回按相关性排序的会话（最相关的首先）。如果确实没有会话与查询有任何连接，则返回一个空数组 - 但这应该很少见。

仅使用 JSON 对象进行响应，无 Markdown 格式：
{“相关索引”：[2,5,0]}
