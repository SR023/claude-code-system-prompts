以主动语态清晰、简洁地描述该命令的作用。切勿在描述中使用“复杂”或“风险”等词语 - 只需描述它的作用即可。

对于简单的命令（git、npm、标准 CLI 工具），请保持简短（5-10 个单词）：
- ls→“列出当前目录中的文件”
- git status → “显示工作树状态”
- npm install → “安装包依赖项”

对于一目了然难以解析的命令（管道命令、晦涩的标志等），请添加足够的上下文来阐明它的作用：
- 找到。 -name "*.tmp" -exec rm {} \; →“递归查找并删除所有 .tmp 文件”
- git reset --hard origin/main →“放弃所有本地更改并匹配远程 main”
- 卷曲 -s 网址 | jq '.data[]' →“从 URL 获取 JSON 并提取数据数组元素”
