# 梦想：记忆巩固

你正在做一个梦——对你的记忆文件进行反思。将您最近学到的知识综合成持久的、组织良好的记忆，以便将来的课程能够快速定位。

内存目录：`ZXQMASK0000QXZ`
${MEMORY_DIR_CONTEXT}

会话记录：`ZXQMASK0002QXZ`（大型 JSONL 文件 - grep 狭义，不要读取整个文件）

---

## 第一阶段——东方

- `ls` 内存目录看看已经存在什么
- 读取`ZXQMASK0003QXZ`了解当前索引
- 浏览现有主题文件，以便改进它们而不是创建重复项
- 如果 `logs/` 或 `sessions/` 子目录存在（助手模式布局），请查看其中最近的条目

## 第 2 阶段 — 收集最近的信号

寻找值得坚持的新信息。按粗略优先顺序排列的来源：

1. **每日日志** (`logs/YYYY/MM/YYYY-MM-DD.md`) 如果存在 - 这些是仅附加流
2. **现有的记忆漂移**——与你现在在代码库中看到的内容相矛盾的事实
3. **转录本搜索** — 如果您需要特定上下文（例如，“昨天构建失败的错误消息是什么？”），则 grep JSONL 转录本以获取狭义术语：
   `grep -rn "<narrow term>" ZXQMASK0004QXZ/ --include="*.jsonl" | tail -50`

不要详尽地阅读文字记录。只寻找你已经怀疑重要的事情。

## 第三阶段——巩固

对于每件值得记住的事情，在内存目录的顶层写入或更新内存文件。使用系统提示符的自动记忆部分中的内存文件格式和类型约定 - 这是要保存什么、如何构建它以及不保存什么的事实来源。

重点关注：
- 将新信号合并到现有主题文件中，而不是创建近似重复的信号
- 将相对日期（“昨天”、“上周”）转换为绝对日期，以便在时间过去后它们仍然可以解释
- 删除相互矛盾的事实——如果今天的调查反驳了旧记忆，请从源头上解决它

## 第 4 阶段 — 修剪和索引

更新 `ZXQMASK0005QXZ`，使其保持在 ${INDEX_MAX_LINES} 行下方且约 25KB 以下。它是一个 **索引**，而不是转储 — 每个条目应该是约 150 个字符下的一行：`- [Title](file.md) — one-line hook`。切勿将内存内容直接写入其中。

- 删除指向现已过时、错误或被取代的内存的指针
- 降级详细条目：如果索引行超过约 200 个字符，则它携带属于主题文件的内容 - 缩短行，移动细节
- 添加指向新重要记忆的指针
- 解决矛盾 - 如果两个文件不一致，请修复错误的文件
