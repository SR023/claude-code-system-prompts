# 简化：代码审查和清理

检查所有更改的文件的重用性、质量和效率。修复发现的任何问题。

## 第 1 阶段：识别变更

运行 `git diff`（如果有分阶段更改，则运行 `git diff HEAD`）以查看更改的内容。如果没有 git 更改，请查看用户提到的或您之前在此对话中编辑的最近修改的文件。

## 第 2 阶段：并行启动三个审核代理

使用 ${AGENT_TOOL_NAME} 工具在一条消息中同时启动所有三个代理。向每个代理传递完整的差异，以便它具有完整的上下文。

### 代理 1：代码重用审查

对于每个更改：

1. **搜索可以替换新编写的代码的现有实用程序和帮助程序**。在代码库的其他地方寻找类似的模式 - 常见位置是实用程序目录、共享模块以及与更改的文件相邻的文件。
2. **标记任何与现有功能重复的新函数。** 建议改用现有函数。
3. **标记任何可以使用现有实用程序的内联逻辑** - 手动字符串操作、手动路径处理、自定义环境检查、临时类型防护和类似模式是常见的候选模式。

### 代理 2：代码质量审查

查看 hacky 模式的相同更改：

1. **冗余状态**：复制现有状态的状态，可以派生的缓存值，可以直接调用的观察者/效果
2. **参数蔓延**：向函数添加新参数，而不是泛化或重组现有参数
3. **略有变化的复制粘贴**：应该与共享抽象统一的近乎重复的代码块
4. **泄漏抽象**：暴露了应该封装的内部细节，或者打破了现有的抽象边界
5. **字符串类型代码**：使用原始字符串，其中常量、枚举（字符串联合）或品牌类型已存在于代码库中
6. **不必要的 JSX 嵌套**：不添加布局值的包装框/元素 - 检查内部组件道具（flexShrink、alignItems 等）是否已提供所需的行为
7. **不必要的注释**：解释代码的作用的注释（命名良好的标识符已经做到了这一点），叙述更改，或引用任务/调用者 - 删除；只保留不明显的原因（隐藏的约束、微妙的不变量、解决方法）

### 代理 3：效率审查

查看相同的更改以提高效率：1. **不必要的工作**：冗余计算、重复文件读取、重复网络/API调用、N+1模式
2. **错过并发**：独立操作在可以并行运行时按顺序运行
3. **热路径膨胀**：新的阻塞工作添加到启动或每个请求/每个渲染热路径
4. **重复无操作更新**：轮询循环、间隔或无条件触发的事件处理程序内的状态/存储更新 - 添加更改检测防护，以便下游消费者在没有任何更改时不会收到通知。另外：如果包装器函数采用更新器/减速器回调，请验证它是否支持相同引用返回（或任何“无更改”信号）——否则调用者的早期返回无操作将被悄悄击败
5. **不必要的存在性检查**：操作前预先检查文件/资源存在性（TOCTOU反模式）——直接操作并处理错误
6. **内存**：无界数据结构、缺少清理、事件侦听器泄漏
7. **过于宽泛的操作**：当只需要一部分时读取整个文件，当过滤一个项目时加载所有项目

## 第三阶段：修复问题

等待所有三个代理完成。汇总他们的发现并直接解决每个问题。如果某个发现是误报或不值得解决，请记下它并继续 - 不要与该发现争论，只需跳过它。

完成后，简要总结修复的内容（或确认代码已经干净）。
